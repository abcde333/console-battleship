# Battleship (Морской бой)

Консольная реализация игры «Морской бой» на Python.

Проект позволяет играть **человеку против бота**.
Игра полностью проходит в консоли, а все ходы и состояния игровых полей сохраняются в CSV-файлы в папке `data`.

Игрок вводит координаты в **графическом формате (A1–J10)**, как в классической игре.

---

## Структура проекта

```
python_proj/
│
├── main.py                  # Точка входа в игру
│
├── src/
│   ├── utils.py              # Работа с координатами, проверка границ, соседи
│   ├── ship_input.py         # Ввод и проверка кораблей игрока
│   ├── bot_generation.py     # Генерация кораблей бота

├── data/
│   ├── player_ships.csv      # Корабли игрока (A1, B2 и т.д.)
│   ├── bot_ships.csv         # Корабли бота
│   └── game_state.csv        # История ходов и состояния полей
│
├── requirements.txt
└── README.md
```

## Запуск игры

1. Установите Python версии **3.8 или выше**
2. (Если требуется) установите зависимости:

```bash
pip install -r requirements.txt
```

3. Запустите игру:

```bash
python3 main.py
```

---

## Управление и ввод

* Все координаты вводятся **в графическом формате**:

  ```
  A1
  F4
  J10
  ```
* Буквы `A–J` — строки поля
* Числа `1–10` — столбцы поля

---

## Правила игры

* Используется классическое поле **10×10**
* Набор кораблей стандартный:

  * 1×4
  * 2×3
  * 3×2
  * 4×1
* Корабли не могут соприкасаться, **даже по диагонали**
* Побеждает тот, кто первым уничтожит все корабли противника

---

## Обозначения на поле

* `S` — корабль
* `X` — попадание
* `M` — промах
* `~` — неизвестная / пустая клетка

---

## Хранение данных

После каждого хода:

* сохраняется ход игрока и бота (в формате A1, F4 и т.д.)
* сохраняется результат хода (попадание, промах, уничтожение корабля)
* сохраняется **состояние обоих полей в текстовом виде**, как в консоли

Все данные записываются в файл:

* `data/game_state.csv`

Это позволяет:

* отследить ход игры
* проверить корректность логики
* восстановить состояние игры

---

## Особенности реализации

* Внутри программы координаты переводятся в индексы `(row, col)`
* Для пользователя и CSV используется **только графический формат**
* Проверка соседних клеток учитывает диагонали
* Бот делает случайные, но корректные ходы
